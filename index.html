1. Nanami and Toys Buying Problem

Description

给定 $n$ 列，有无限行，第一行与第二行的物品价值用数组 $a, b$ 给出，第二行以下的物品价值与第二行相同，给定 $k$ 次机会取走，每次取必须取每列最上面的物品，问最大可取价值是多少

Solution

现在我们有几种策略，一是每次只拿表面价值最高的物品，二是找到第二行价值最大的物品，拿掉上面的，剩下全部拿该列的物品

但是我们可以通过几个简单的例子证明这些策略是错误的：

(1) 共拿k次
1 100 1000 10000
10000000000 100 1000 1000
(2) 共拿1次
3 4 5
10000 0 0

我们发现策略一没有考虑到第二行，导致不拿1下面的最大数；策略二没有考虑到第一行和k，如果次数有限则应该优先拿第一行最大的

所以我们要想到一个考虑第一行与第二行，同时兼顾k的策略

于是我们想，如果要获得最大的价值，我们应该考虑两个数组中的最大数，如果在第一行，则我们直接拿，如果在第二行，则我们要拿掉上面的才能拿到，此时还要考虑到k是否足够的问题，如果不够，则应该优先选第一行的次大数

但是我们看下面的例子：

共拿3次
0 1000 1001 1002
1003 0 0 0

这个例子中，如果按照我们的策略，则我们会选择 0 1003 1003，而实际上应该选择 1002 1001 1000，这是没有考虑到上面的数对整体的影响

那么我们换一种思路，我们能否通过暴力的方法解决呢

我们假设最终用于累加的列为第i列，则我们可以通过二分查找的方法找第一行中比第二行第i列数大的所有数，判断取完这些数后是否还有次数，如果有就看这些数里面有没有第i列的第一行数，有就直接取第二行，没有就取完第一行再取第二行，然后计算价值，最后取这些价值中的最大值即为解

而由于我们在查找过程中使用了二分，复杂度为 $O(n\log{n})$ ，是在可接受的范围内

Details

  1. 本题需要特判最大值在第二行的情况，因为二分查找会查到第一行的最大值，但是最大值不取第一行最大值，而是直接取第二行的最大列

11. 安营扎寨

Description

求最大子矩阵和，但是要求子矩阵必须包含最中间的四个点

Solution

基于最大子段和，求最大子矩阵和升级为二维情况，这时需要考虑的因素变多

我们可能会想尝试二维情况下的动态规划，依照一维的情况，假设 $dp[i][j]$ 是以 $(i,j)$ 为右下端点的最大子矩阵和，但是在设计转移方程的时候发现需要左上角端点的信息，而且需要 $(i-1,j),(i,j-1)$ 两个点的信息，这使得难以写出状态转移方程

其实更好的办法是将二维的问题转化为一维的问题，然后枚举求解

我们用 $sum[i][j]$ 来表示第 $j$ 列前 $i$ 行数的和，然后给定上下界 $u,d$ ，转化为求 $a[j] = sum[d][j] - sum[u-1][j]$ 这个数列的最大子段和（其实就是将 $[(u, 1), (d, m)]$ 这个区域的列求和后作为数组求最大子段和）

我们只需要枚举 $u,d$ 的组合就可以了

由于一维的最大子段和复杂度为 $O(m)$ ，而枚举的复杂度为 $O(n^2)$ ，所以本题做法的复杂度为 $O(mn^2)$

Details

  1. 本题会出现极端情况，即 $m = 1, n = 10^5$ 此时需要掉换 $m,n$ 位置，如此控制后，复杂度变为 $O(mn \max (m, n))$

14. 小L的神奇修正带

Description

给定两个字符串 $S,T$ 其中 $|S| \ge |T|$ ，问 $S$ 是否为 $T$ 不断叠加覆盖形成的

Solution

首先我们要注意到一件事情：如果 $S$ 是叠加而来，无论 $T$ 怎么叠加，最后叠加的一定在最表面，即给定的 $S$ 表面一定有至少一个 $T$

我们将其“撕掉”，但是剩下空位怎么处理呢？

我们并不知道底下的具体叠加情况，但是我们可以假设底下的叠加都是合法的，这些空位可以随便指定一个占位符，在这种情况下，我们继续查找“表面”的 $T$，只不过这一次，我们把占位符看成通配符，即这些字符匹配任何字符都是合法的

最后我们只需要检测 $S$ 字符串是否全为占位符就可以判断答案

这样的复杂度为 $O(mn)$ 大约为 $5 \times 10^5$ 的量级，所以不用担心会TLE

Details

  1. 由于我们是一层一层“撕”的，所以我们使用队列来完成这个任务
  2. 不要重复添加已经撕下的部分，即全是占位符是不允许的

17. 汽水装填

Description

给定一个数组的初始状态，再给定操作区间与操作数，把区间内小于操作数的数全部设置成为操作数，其余数不变，最后输出数组的最终状态

Solution

这种涉及到区间操作的一般有多种选择，比如线段树、树状数组等等，这里使用线段树

线段树使用二叉树结构，每个节点表示一段子区间，而把该节点的区间分成两半，左边区间赋给左子节点，右边的区间赋给右子节点，一直递归到区间中只有一个数

同时，每个节点含有一个标记，表示对整个区间的操作数，就本题而言，其表示区间中最大的操作数

以下是具体的线段树的操作过程：

给定一个区间 $[l,r]$ 与操作数 $b$ ，对于某一个节点来说有三种情况

  • 节点表示的区间与目标区间没有交集，这时直接返回
  • 节点表示的区间被包含在目标区间中，这时将其标记 $m$ 设置为 $\max (m, b)$ ，然后返回
  • 节点表示的区间与目标区间相交，但是不是包含关系，这时需要递归到左右子节点进行操作，但是在调用左右节点前，需要把自己的标记 $m$ “传”到左右子节点 $m{child} = \max (m, m{child})$ ，然后把自己的标记设0后再进行调用

之所以要“传”标记，是因为有可能左右节点区间的情况不一样，需要把标记传下去，让左右节点依据自己情况具体判断，同时父节点设0防止干扰

下面是查询线段树的操作过程：

给定查询区间 $[l,r]$ （一个点也可以写成区间的形式），对于某一个节点依旧是上面的三种情况：

  • 没有交集，返回0
  • 包含关系，返回标记
  • 相交关系，“传标记”，查询左右节点，返回两者之间的最大值

最后输出的状态就是 $\max (a_i, m_i)$

单次操作的时间复杂度为 $O(\log n)$ ，单次查询的复杂度为 $O(\log n)$

Details

  1. 就本题而言，cin与cout需要关闭同步流，或者使用pringf与scanf，否则会TLE

Modern C++: move && forward
2023/12/22

std::move()

该函数将传入的参数变成将亡值（会清空该参数），同时返回该函数的右值，方便移动构造，实现所有权交接

其本质并不是移动，而是强制类型转换，把参数强制转换成为右值

ps. 函数返回局部对象时请不要 std::move ，你的编译器会自动做优化 (RVO) ，如果使用可能会破坏优化

std::forward()

problem

在使用万能引用时，由于不清楚最终参数会被匹配为什么引用，而 C++ 中，所有引用都是左值，当外部传入一个右值时，在模板内部的参数是右值引用，当在模板内部传入该引用，其会被认为是左值，从而匹配左值引用，所有需要 forward 进行转发

function

std::forward()实现了完美转发（即正确调用函数），左值引用传左值引用的重载函数，右值引用传右值引用的重载函数

其本质是有选择的类型转换，把右值引用转化为右值，左值引用则保持不变

其原理其实是 T&& 中的 T 保存了传入时的参数状态：左值或者右值

difference

std::forward() 有选择的右值转换

std::move() 强制右值转换

Modern C++: Lvalue && Rvalue
2023/12/22

左值、右值、将亡值

左值

左值（lvalue）是可以修改的值，是在可修改区域（堆、栈、全局）的值，是有名字的值

ps. 所有的函数形参都是左值

右值

右值（rvalue）是不可以修改的值，是在不可修改区域（代码区）的值，是无名的值

将亡值

将亡值（xvalue）是生命周期很短，即将释放的值

三种引用

引用本质是对指针的封装，是C++对地址空间的保护

左值引用

左值引用是给地址空间起别名，是对地址的映射

右值引用

右值引用是将不可修改的右值拷贝一份放到可修改区域，并为该地址取得名，是该地址的映射，所以右值引用是左值

当右值引用作为函数形参时，只能匹配右值，这通常用来实现移动语义

万能引用

万能引用是 C++ 模板编程中的特殊引用，其既可以匹配左值，又可以匹配右值，所以称为万能引用

基于引用折叠，左值会匹配为左值引用，右值会匹配为右值引用

在模板内部，当需要调用其他函数并且传入万能引用形参时，需要对万能引用调用forward进行完美转发

Solution

本题是经典的动态规划题——最大字串和的变种，先介绍一下正常最大字串和的动态规划做法

The Maximal Sum Of The Substr (Kadane)

给定一个数组，找到最大的字串的和
动态规划三件套——初始条件、状态、状态转移方程
这里状态可以设置：第 $i$ 位之前以第 $i$ 位为终点的最大字串和
初始条件：第0位就是 $a[0]$ 的值
状态转移方程：
解释：
我们假设第 $i-1$ 位前最大字串是以第 $i-1$ 位为结尾的字串，则对于第 $i$ 位来说，如果前面的字串和加上自己还不如自己本身的值大（说明前面的字串和为负数），则最大的字串就是自己本身；否则，就把自己加入到字串中，形成更大的字串
这样，对于每一位 $i$ 来说，我们通过动态规划考虑了其所有字串，所以我们只要遍历 $dp[i]$ ，我们就考虑到了所有字串，进而我们可以找到最大值

而对于本题而言，我们只需在最大字串和的基础上增加判断条件即可，即前后两个数的奇偶性是否相同，相同就取自己为字串，不同就进行正常判断

    F

    Description

    https://codeforces.com/contest/1883/problem/F

    Solution

    题意要求找到不重复的子列的数量，但是不要求子列是连续的
    本题关键在于如何找到不重复的子列，同D，两个子列不重复的条件是什么？不好判断，所以我们想两个子列重复的条件是什么？
    两个子列的左右边界上的数相等
    但是这是一个必要条件，我们如何补充条件使其充分？
    如果子列的左边界左边没有左边界上的数，右边界右边没有右边界上的数，则该子列一定是不重复的
    可以证明，这样取子列可以取完所有符合条件的子列
    假设对于子列$[l,r], \exist \alpha \in [0, l],\beta \in [r,n-1],a[\alpha]=a[l],a[\beta]=a[r]$，则我们可以取出$[l+1,r]$和$[l,r-1]$的子列组成两个同样子列，矛盾！
    所以我们有了一个贪心策略
  • 首先生成两个数列，分别记录该位置上的数是否是第一次出现和最后一次出现
  • 然后对第一次出现的数列求前缀和
  • 遍历每一个最后一次出现的数，将其对应的前缀和相加

    G

    Description

    https://codeforces.com/contest/1883/problem/G1
    https://codeforces.com/contest/1883/problem/G2

    Solution

    本题分为两个难度，后一个是前一个的深化

    G1

    解题的关键就在于数列排序的任意性，既然排序是任意的，不妨将两个数列排序后再进行操作
    由于我们想尽可能减少移除的元素个数，所以我们应该建立尽可能多的映射$a[i] \lrArr b[j], a[i] < b[j]$，由此我们有了一个贪心策略：
  • 对于每一个$a[i]$，向后遍历$b[j]$直到找到严格大于$a[i]$的$b[j]$
  • 找到数组结尾就结束

直观上看，因为$b[j]$越往后越大，跟田忌赛马一样，应该能对应更多的$a[i]$，于是我们有了G1的双指针+贪心的做法：

  • 两个指针分别指向$a[0]$和$b[0]$
  • 如果$b[j]$不是严格大于$a[i]$，就让$b$指针向后移动直到找到严格大于的数
  • 如果满足映射关系，则两个指针同时自增

    G2

    接下来的题目在$a[0]$从1到m的条件下求G1，并且输出所有答案的和
    如何减少时间复杂度呢？这依赖于一个需要观察的性质
    假设$f(x)$表示$a[0] = x$的条件下的解，则可以由归纳法证明：
    因为变化的只有$a[0]$，所以不考虑$a[0]$，在$a[i],i \geq 1$与$b[j]$建立起映射，因为排序是任意的，取剩下的$b[j]$的最大值$M$，如果$a[0]<M$，则建立起映射，$f(a[0])$就不变；反之，则$f(a[0])$必须要多删一个
    可知$k$就是剩下$b[j]$中最大值的下标
    所以我们可以只用求一次G1，然后通过找出$k$，经过判断和简单的四则运算就可以输出答案，大大降低了时间复杂度

    Note

  1. G2会爆int
  2. 因为不管$a[0]$，所以要在排序后给它赋值
  3. 不要用iterator，会有莫名其妙的Access Violation
  4. any element是任意一个元素，不是任选元素

Description

Solution

这道题最大的难点在于把$l$与$r$数组的特殊性，以及$a$和$b$操作的特殊性弄清楚

$l$ & $r$

题目说给一个x夹在某个$l$与$r$中间，意思就是$l$与$r$是区间$[l,r]$的集合，而且这个集合拥有以下的特性：

  • $l_1$对应字符串开头
  • $r_n$对应字符串结尾
  • 两个区间是紧挨着的（$l_{k+1}=r_k+1$）

所以可见这个集合覆盖了整个字符串，而且两两之间不重合

$a$ & $b$

题面给的$a$和$b$经过观察样例可以发现，$a$和$b$在某个区间内是左右对称的

思路

所以我们可以采取用数组记录操作的位置，然后在输出的时候根据记录来决定移动的方向和读取的顺序
数组如此记录$arr[a]=b, arr[b]=a$
读取如下：

  • 如果$arr[i]=-1$，则输出
  • 否则跳转至$arr[i]$的位置，按相反方向输出
  • 途中如果遇见$arr[i]$有值的情况，则跳转然后反向输出
  • 直到输出到跳转起始位置

    E

    Description

    https://codeforces.com/contest/1878/problem/E

    Solution

    本题的关键在于如何快速的求出$f(l,r)$，以及如何快速地找到$r$

    Calculate $f(l, r)$

    由&的性质，一旦$a_i$的某一位为0，则之后的操作过后，该位还是0
    我们记录一个二维数组$arr[i][j]$表示直到第$i$个数（包括），第$j$位0的总数，所以对于$f(l, r)$，我们只需要知道$a_l$和$arr[l][j], arr[r][j]$的值就可以快速算出
    （$arr[l][j] \neq arr[r][j]$表示&后的第$j$位是0）

    Find $r$

    由于&的性质，$a \geq a \And b$，所以可以认为在$l$固定的情况下，$f(l, r)$关于r递减，可以看作是一个降序的数组
    降序的数组怎么查最快？当然是二分查找！

    思路

    所以我们在读取数据后生成对应的$arr$，然后按照以上方法进行计算即可

    Binary Search

    Main Structure

    二分查找一定有$l$和$r$，然后根据条件不断二分

    int l = begin, r = end;
while (l < r) {
  int mid = (l + r) / 2;
  if (condition) statement1;
  else statement2;
}

    Details

    根据情况不同，判断的条件也不同
    这里假设$arr$是升序的
    一共有两类情况：

  1. 查找大于（等于）$k$的最左的位置
  2. 查找小于（等于）$k$的最右的位置
Index Condition Statement1 Statement2
1 arr[mid] >(=) k r = mid l = mid + 1
2 arr[mid] <(=) k l = mid r = mid - 1

这里1、2的表述其实是一样的，因为查到了1，往左一位就是2的答案
而在实际中，应该尽量使用1而不是2，尽量避免l = mid, r = mid - 1这种情况出现，因为当$r-l=1$的时候，$mid=l$，这就会导致$l=mid$不断执行而不会结束循环，而1的写法就能避免这一点

还需要注意的一点，当将2的情况变为1的写法时，一定要注意边界结果，因为我们返回答案时会返回$l-1$，所以当答案是$size-1$时，会返回$size-2$

Description

https://codeforces.com/contest/1878/problem/F

Solution

这一道就是纯粹的数论题了
当$\gcd(a,n)=1$时，$d(a \times n)=d(a)d(n)$，这可以由数的质因数分解来证明，证明如下：

所以要满足$d(an)=d(n)d(a)=n$需要保证n是1或2或合数
如果n为1，则显然a为1可以满足
如果n为2，则a为1仍然可以满足
如果n是合数，则n还需要满足$d(n)|n$这个性质
那a呢？事实上，a可以取不是n的因数的素数，保证$\gcd(a,n)=1$，同时令$d(a)=\frac{n}{d(n)}$，可以得出$a=p^{\frac{n}{d(n)}-1}$
于是我们得到判据：

  • 如果n为非2的素数，则不满足
  • 如果n为合数而且$d(n)\nmid n$，则不满足
  • 其他情况都满足

    Details

    注意本题数据范围，$n\times x$会很大，甚至会爆unsigned long long，所以要用质因数分解的形式储存和计算

思路

首先最原始的思路就是从$h=1$开始，不断累加，然后判断容量是否比$x$大，直到找到$wh<x<w{h+1}$，当然，当数据加强的时候，一定会TLE
比这个更优的是利用二分法加速，因为我们发现改变数组的顺序不影响结果，那我们用任意一个排序，使数组升序，方便二分，但是当$h>\max{ah}$时，我们必须单独判断，除此之外，因为有可能$h$夹在$a_h$中间，所以二分必须是先找$a_h$和$a{h+1}$，然后再计算高度
综上，步骤为：

  • 读取数据和排序
  • 判断是否$h>\max{a_h}$，如果是，就直接输出结果
  • 如果不是，则开始二分查找$ah<x<a{h+1}$，一旦中间出现等于，立即返回

    注意事项

  1. 由于计算容量的时候要经历$\log(N)$的操作，不妨记录一个前缀和presum，则计算容量只需要每次只需要查表即可
  2. 二分的时候一定要设置$left$和$right$，这是二分的必须条件

    F

    题面

    https://codeforces.com/contest/1873/problem/F

    思路

    这道题是一道经典的尺取法的问题，我们要考虑的问题有两个：
    如何取得连续整除的子列？
    如何保证子列的和不超过$k$?
    既然是尺取法，则需要双指针，那一开始指针就必须指向一个$ai<k$的数
    按题意，首先必须满足$h    {i}|h{i-1}$，然后判断加上$a_i$会不会超$k$，没有就记录长度，超了就让前面的一端向后移动直到不超位置
    如果不满足$h    {i}|h_{i-1}$，则让前端从后端开始寻找下一个$a_i<k$，重置初始条件，重复以上过程
    综上，步骤为：
  • 循环找到启示指针位置
  • 开始移动指针，判断$h{i}|h{i-1}$
    • 满足，判断加上$a_i$超不超，超了就移动前指针找到不超的$a_i$；最后记录最长长度
    • 不满足，前指针到后指针处，循环找到合适位置，重置和与后指针

      注意事项

      在寻找合适位置的时候要保证前指针小于后指针，如果发现一直找不到合适位置，直接返回已有长度

      H

      题面

      https://codeforces.com/contest/1873/problem/H

      思路

      本题面临以下几个问题：

      怎样判断能否一直逃脱

      根据题意，n个节点却只有n条边，所以图中只能且必须有一个环。
      只有当b能到达环，且a不能在b到达环之前抓住b，b才能逃脱
      这衍生出了几个问题：
  1. 如何找出环
  2. 如何判断b到环的距离
  3. 如何判断a能否在b到环上前抓住b

    如何找出环

  • 利用并查集，在读取边的时候，如果发现两个顶点已经在同一个集合内了，则说明它们一定是环上的两个点
  • 从这环上的两个点其中之一出发，深度优先搜索dfs另一个点
  • 搜索到终点后，做一个全局标记，让函数返回上一层时记录下对应的点，得到所有环上的点
  • 最后将两个点连起来

    如何判断b到环的距离

  • 利用两个数组，一个表示b到图中任意一点的距离，另一个表示节点是否访问过
  • 从b出发，广度优先搜索bfs，记录下沿途每一个点的距离，标记访问
  • 如果搜索到环上的点，记录距离和该点的编号（不妨设为c），停止；

    如何判断a能在b到环上前抓住b

    只要a到c的距离小于等于b，则a一定能抓住b，则同上：
  • 还是两个数组
  • 从a出发，bfs，记录距离和标记访问
  • 直到搜索到c
