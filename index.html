C

题面

https://ac.nowcoder.com/acm/contest/65157/C

解题思路

本题的目的是找到第k小的数的最小值，其实就是通过以上规则生成k大小的升序数组arr，且保证数组内的任意两个数的和要么比arr[k]大，要么就是arr中的数，不可能有新的数能够插入到arr中
那我们的思路就是从小往大生成数组

暴力生成

对于已有的数组arr，我们直接将所有可能的和放入堆中，最后输出最小的那个，放到arr最后
有几个细节：

  • 在加入heap之前一定要检查是否重复
  • arr加入heap的堆顶元素也要检查是否重复

动态规划

前面暴力生成的时间复杂度为O(k!)，显然会超时
考虑数组中的每一个元素，其都是由给定的a和b生成而来，即
$$a_i=ma+nb$$
所以想要找到数组的下一个数，即找到
$$a_{k+1}=\min(a_i+a_j)=\min((m_i+m_j)a+(n_i+n_j)b)>a_k$$
又有
$$\min(a_i+a_j)=\min(a_j+a, a_j+b)$$
故对于a和b，我们分别用两个指针去寻找它们最小临界的位置，即
$$a_{p_1} = \min(a_j+a) > a_k$$
$$a_{p_2} = \min(a_j+b) > a_k$$
然后再取两者的最小值作为数组的下一个数

有几个细节：

  • 一定是两个指针，不能只取最小值的指针，如a = 3，b = 10，在生成到{3, 6, 9, 10, 12, 13, 15, 16, 18, 19}时，如果按最小值指针，则会生成21，但是取10+10生成20，显然不对

D

题面

https://ac.nowcoder.com/acm/contest/65157/D

解题思路

法一

从简单入手，如果是满二叉树，则按树的高度H依次计数，假设有n个节点
$$n=2^H-1$$
树高为i时有满二叉树数量为
$$a_i=2^{H-i+1}-1$$
求和有
$$sum = \sum_{i=1}^H a_i = 2^{H+1}-H-2$$
如果增加k个节点(k < 2^H)，则对应的表为

k 1 2 3 … $2^{H}-1$
增加的值 1 3 4 … $2^{H+1}-H-2$
k的二进制 1 10 11 … 111…111

观察发现增加的值可以这样计算

  • k的二进制中为1的位置pos
  • 计算$2^{pos}-1$
  • 所有的这些数加起来即为增加的值

法二

对于每一层而言，有$a_i$个数
当只计算树高为1时，它们的贡献为$a_i$
当只计算树高为2时，它们的贡献为$a_i/2$
当只计算树高为3时，它们的贡献为$a_i/4$
…
当只计算树高为k时，它们的贡献为$a_i/2^{k-1}$
最后所有贡献加和即为该层的总贡献

细节处理

因为最后的答案可能很大所以要在每一步都要模$10^9+1$

long long solve2(long long n) {
    int H = 1;
    long long sum = 0;
    while (n > 0) {
        long long t;
        if (n >= ((long long)1 << (H - 1))) {
            t = ((long long)1 << (H - 1));
        }
        else {
            t = n;
        }
        n -= t;
        while (t > 0) {
            sum += t % MOD; //not enough
            sum %= MOD; //the essence!!!
            t >>= 1;
        }
        H++;
    }
    return sum;
}

E

题面

https://ac.nowcoder.com/acm/contest/65157/E

解题思路

法一

题意非常明确，思路按照题意来，唯一的难点是如何判断是否死循环
这里采用set，在每次操作完一轮后将x插入set中，如果重复则说明死循环

法二

按照法一来，会遇见时间复杂度过大的问题，从而导致RE
这里将次数分为两个部分，a轮+b步，先讨论一轮之后的情况，假设$x_k$表示k轮后的x，由于
$$x=(x+a_{(i-1)mod(m)})mod(n)$$
可以认为
$$x_{k+1}=(x_k+\sum_{i=1}^ma_i)mod(n)$$
故$x_k$是模n的一个剩余类
可知，由抽屉原理，在$n+1$轮中，一定有两个x相等，故循环最多进行n轮，n轮中没有0则说明死循环
再讨论k步后的情况，假设k步后恰好x为0，则说明
$$x+(\sum_{i=1}^ka_i)mod(n)=n$$
所以我们可以用一个map记录前缀和对n取模的值（如果有两个值相等，保留最小的下标），每轮只需要查询$n-x$在map中是否存在就行了

注意事项

这道题会爆int，必须用long long

C

Description

https://codeforces.com/contest/1899/problem/C

Solution

本题是经典的动态规划题——最大字串和的变种，先介绍一下正常最大字串和的动态规划做法

The Maximal Sum Of The Substr (Kadane)

给定一个数组，找到最大的字串的和
动态规划三件套——初始条件、状态、状态转移方程
这里状态可以设置：第 $i$ 位之前以第 $i$ 位为终点的最大字串和
初始条件：第0位就是 $a[0]$ 的值
状态转移方程：$$dp[i] = \max { a[i], a[i] + dp[i-1] }$$
解释：
我们假设第 $i-1$ 位前最大字串是以第 $i-1$ 位为结尾的字串，则对于第 $i$ 位来说，如果前面的字串和加上自己还不如自己本身的值大（说明前面的字串和为负数），则最大的字串就是自己本身；否则，就把自己加入到字串中，形成更大的字串
这样，对于每一位 $i$ 来说，我们通过动态规划考虑了其所有字串，所以我们只要遍历 $dp[i]$ ，我们就考虑到了所有字串，进而我们可以找到最大值

而对于本题而言，我们只需在最大字串和的基础上增加判断条件即可，即前后两个数的奇偶性是否相同，相同就取自己为字串，不同就进行正常判断

Solution

本题是构造类型的题，其要求不断构造树来满足叶节点距离
但实际上，我们观测样例可以知道，一开始，其构造的树都是一条链，根据后续的 $d$ 来改变树的形状
一开始，我们只有头和尾两个叶节点，为了满足需求，我们需要改变叶节点之间的距离，怎么又快又好地改变叶节点的距离，那就是通过只移动一个固定节点的位置（比如尾节点），来改变它和其他叶节点的距离（比如头节点）
比如我们有一条链 $[1,2,3,4,5,6]$ ，我们现在需要距离为3，我们就只需要把6号与5号断开，连上3号就可以了，操作数为 $6, 5, 3$
一旦意识到这点，这题就非常简单了。

现在我们思考，两个区间不重叠的条件是什么？

Description

https://codeforces.com/contest/1883/problem/E

Solution

题意要求通过乘2操作使原数列变为非减数列
假设经过操作后的第$i$位变为$2^{\alpha}a_i$，第$i+1$位变为$2^{\beta}a_{i+1}$，则应该满足
$$2^{\alpha}a^i \leq 2^{\beta}a_{i+1}$$
$$\beta \geq \log_2(\frac{2^{\alpha}a^i}{a_{i+1}})$$
通过向上取整可以得到每一位的最小操作数

Note

  1. 本题会爆unsigned long long，所以要单独存储2的指数
  2. 本题要注意log的运算优化，防止精度损失

Description

https://codeforces.com/contest/1883/problem/F

Solution

题意要求找到不重复的子列的数量，但是不要求子列是连续的
本题关键在于如何找到不重复的子列，同D，两个子列不重复的条件是什么？不好判断，所以我们想两个子列重复的条件是什么？
两个子列的左右边界上的数相等
但是这是一个必要条件，我们如何补充条件使其充分？
如果子列的左边界左边没有左边界上的数，右边界右边没有右边界上的数，则该子列一定是不重复的
可以证明，这样取子列可以取完所有符合条件的子列
假设对于子列$[l,r], \exist \alpha \in [0, l],\beta \in [r,n-1],a[\alpha]=a[l],a[\beta]=a[r]$，则我们可以取出$[l+1,r]$和$[l,r-1]$的子列组成两个同样子列，矛盾！
所以我们有了一个贪心策略

  • 首先生成两个数列，分别记录该位置上的数是否是第一次出现和最后一次出现
  • 然后对第一次出现的数列求前缀和
  • 遍历每一个最后一次出现的数，将其对应的前缀和相加

G

Description

https://codeforces.com/contest/1883/problem/G1
https://codeforces.com/contest/1883/problem/G2

Solution

本题分为两个难度，后一个是前一个的深化

G1

解题的关键就在于数列排序的任意性，既然排序是任意的，不妨将两个数列排序后再进行操作
由于我们想尽可能减少移除的元素个数，所以我们应该建立尽可能多的映射$a[i] \lrArr b[j], a[i] < b[j]$，由此我们有了一个贪心策略：

  • 对于每一个$a[i]$，向后遍历$b[j]$直到找到严格大于$a[i]$的$b[j]$
  • 找到数组结尾就结束

直观上看，因为$b[j]$越往后越大，跟田忌赛马一样，应该能对应更多的$a[i]$，于是我们有了G1的双指针+贪心的做法：

  • 两个指针分别指向$a[0]$和$b[0]$
  • 如果$b[j]$不是严格大于$a[i]$，就让$b$指针向后移动直到找到严格大于的数
  • 如果满足映射关系，则两个指针同时自增

G2

接下来的题目在$a[0]$从1到m的条件下求G1，并且输出所有答案的和
如何减少时间复杂度呢？这依赖于一个需要观察的性质
假设$f(x)$表示$a[0] = x$的条件下的解，则
$$\exist k, s.t. f(1)=f(2)=\cdots=f(k)=f(k+1)-1=\cdots=f(\infty)-1$$
可以由归纳法证明：
因为变化的只有$a[0]$，所以不考虑$a[0]$，在$a[i],i \geq 1$与$b[j]$建立起映射，因为排序是任意的，取剩下的$b[j]$的最大值$M$，如果$a[0]<M$，则建立起映射，$f(a[0])$就不变；反之，则$f(a[0])$必须要多删一个
可知$k$就是剩下$b[j]$中最大值的下标
所以我们可以只用求一次G1，然后通过找出$k$，经过判断和简单的四则运算就可以输出答案，大大降低了时间复杂度

Note

  1. G2会爆int
  2. 因为不管$a[0]$，所以要在排序后给它赋值
  3. 不要用iterator，会有莫名其妙的Access Violation
  4. any element是任意一个元素，不是任选元素

Description

https://codeforces.com/contest/1878/problem/D

Solution

这道题最大的难点在于把$l$与$r$数组的特殊性，以及$a$和$b$操作的特殊性弄清楚

$l$ & $r$

题目说给一个x夹在某个$l$与$r$中间，意思就是$l$与$r$是区间$[l,r]$的集合，而且这个集合拥有以下的特性：

  • $l_1$对应字符串开头
  • $r_n$对应字符串结尾
  • 两个区间是紧挨着的（$l_{k+1}=r_k+1$）

所以可见这个集合覆盖了整个字符串，而且两两之间不重合

$a$ & $b$

题面给的$a$和$b$经过观察样例可以发现，$a$和$b$在某个区间内是左右对称的

思路

所以我们可以采取用数组记录操作的位置，然后在输出的时候根据记录来决定移动的方向和读取的顺序
数组如此记录$arr[a]=b, arr[b]=a$
读取如下：

  • 如果$arr[i]=-1$，则输出
  • 否则跳转至$arr[i]$的位置，按相反方向输出
  • 途中如果遇见$arr[i]$有值的情况，则跳转然后反向输出
  • 直到输出到跳转起始位置

E

Description

https://codeforces.com/contest/1878/problem/E

Solution

本题的关键在于如何快速的求出$f(l,r)$，以及如何快速地找到$r$

Calculate $f(l, r)$

$$f(l, r)=a_l \And a_{l+1} \And … \And a_r$$
由&的性质，一旦$a_i$的某一位为0，则之后的操作过后，该位还是0
我们记录一个二维数组$arr[i][j]$表示直到第$i$个数（包括），第$j$位0的总数，所以对于$f(l, r)$，我们只需要知道$a_l$和$arr[l][j], arr[r][j]$的值就可以快速算出
（$arr[l][j] \neq arr[r][j]$表示&后的第$j$位是0）

Find $r$

由于&的性质，$a \geq a \And b$，所以可以认为在$l$固定的情况下，$f(l, r)$关于r递减，可以看作是一个降序的数组
降序的数组怎么查最快？当然是二分查找！

思路

所以我们在读取数据后生成对应的$arr$，然后按照以上方法进行计算即可

Binary Search

Main Structure

二分查找一定有$l$和$r$，然后根据条件不断二分

int l = begin, r = end;
while (l < r) {
    int mid = (l + r) / 2;
    if (condition) statement1;
    else statement2;
}

Details

根据情况不同，判断的条件也不同
这里假设$arr$是升序的
一共有两类情况：

  1. 查找大于（等于）$k$的最左的位置
  2. 查找小于（等于）$k$的最右的位置
Index Condition Statement1 Statement2
1 arr[mid] >(=) k r = mid l = mid + 1
2 arr[mid] <(=) k l = mid r = mid - 1

这里1、2的表述其实是一样的，因为查到了1，往左一位就是2的答案
而在实际中，应该尽量使用1而不是2，尽量避免l = mid, r = mid - 1这种情况出现，因为当$r-l=1$的时候，$mid=l$，这就会导致$l=mid$不断执行而不会结束循环，而1的写法就能避免这一点

还需要注意的一点，当将2的情况变为1的写法时，一定要注意边界结果，因为我们返回答案时会返回$l-1$，所以当答案是$size-1$时，会返回$size-2$

F

Description

https://codeforces.com/contest/1878/problem/F

Solution

这一道就是纯粹的数论题了
当$\gcd(a,n)=1$时，$d(a \times n)=d(a)d(n)$，这可以由数的质因数分解来证明，证明如下：
$$设n=p_1^{\alpha_1}p_2^{\alpha_2}\cdots p_r^{\alpha_r}, a=q_1^{\beta_1}\cdots q_s^{\beta_s},其中p_i,q_i都是素数$$
$$\because \gcd(a,n)=1$$
$$\therefore p_i \neq q_j$$
$$由定义可知: d(n)=\prod_{i=1}^r(\alpha_i+1)$$
$$\therefore d(a\times n)=(\alpha_1+1)(\alpha_2+1)\cdots(\alpha_r+1)(\beta_1+1)\cdots(\beta_s+1)=\prod_{i=1}^r(\alpha_i+1)\prod_{i=1}^s(\beta_i+1)=d(n)d(a)$$
所以要满足$d(an)=d(n)d(a)=n$需要保证n是1或2或合数
如果n为1，则显然a为1可以满足
如果n为2，则a为1仍然可以满足
如果n是合数，则n还需要满足$d(n)|n$这个性质
那a呢？事实上，a可以取不是n的因数的素数，保证$\gcd(a,n)=1$，同时令$d(a)=\frac{n}{d(n)}$，可以得出$a=p^{\frac{n}{d(n)}-1}$
于是我们得到判据：

  • 如果n为非2的素数，则不满足
  • 如果n为合数而且$d(n)\nmid n$，则不满足
  • 其他情况都满足

Details

注意本题数据范围，$n\times x$会很大，甚至会爆unsigned long long，所以要用质因数分解的形式储存和计算

题面

https://codeforces.com/contest/1873/problem/E

思路

首先最原始的思路就是从$h=1$开始，不断累加，然后判断容量是否比$x$大，直到找到$w_h<x<w_{h+1}$，当然，当数据加强的时候，一定会TLE
比这个更优的是利用二分法加速，因为我们发现改变数组的顺序不影响结果，那我们用任意一个排序，使数组升序，方便二分，但是当$h>\max{a_h}$时，我们必须单独判断，除此之外，因为有可能$h$夹在$a_h$中间，所以二分必须是先找$a_h$和$a_{h+1}$，然后再计算高度
综上，步骤为：

  • 读取数据和排序
  • 判断是否$h>\max{a_h}$，如果是，就直接输出结果
  • 如果不是，则开始二分查找$a_h<x<a_{h+1}$，一旦中间出现等于，立即返回

注意事项

  1. 由于计算容量的时候要经历$\log(N)$的操作，不妨记录一个前缀和presum，则计算容量只需要
    $$V_{water}=(i+1) \times height[i]-presum[i]$$
    每次只需要查表即可
  2. 二分的时候一定要设置$left$和$right$，这是二分的必须条件

F

题面

https://codeforces.com/contest/1873/problem/F

思路

这道题是一道经典的尺取法的问题，我们要考虑的问题有两个：
如何取得连续整除的子列？
如何保证子列的和不超过$k$?
既然是尺取法，则需要双指针，那一开始指针就必须指向一个$a_i<k$的数
按题意，首先必须满足$h_{i}|h_{i-1}$，然后判断加上$a_i$会不会超$k$，没有就记录长度，超了就让前面的一端向后移动直到不超位置
如果不满足$h_{i}|h_{i-1}$，则让前端从后端开始寻找下一个$a_i<k$，重置初始条件，重复以上过程
综上，步骤为：

  • 循环找到启示指针位置
  • 开始移动指针，判断$h_{i}|h_{i-1}$
    • 满足，判断加上$a_i$超不超，超了就移动前指针找到不超的$a_i$；最后记录最长长度
    • 不满足，前指针到后指针处，循环找到合适位置，重置和与后指针

注意事项

在寻找合适位置的时候要保证前指针小于后指针，如果发现一直找不到合适位置，直接返回已有长度

思路

这道题是一道动态规划的题
由题意可以知道，B是一个关键的结算节点，B前面或者是后面的A的数量决定着这个B可以进行的操作次数，那不妨使用一个数组记录每个B前后的A的数量
状态设置
假设$dp[i][j],(j \in {0, 1})$表示直到第$i$个B选择$j$状态的累计最大操作数，0表示选择前面，1表示选择后面
初始状态为
$$dp[0][0]=B[0][0], dp[0][1]=B[0][1]$$
状态转移
$$dp[i][0]=\max{dp[i-1][0]+B[i][0],dp[i-1][1]}$$
$$dp[i][1]=\max{dp[i-1][0]+B[i][1],dp[i-1][1]+B[i][1]}$$
最后取$dp$中最大值即可

思路

本题面临以下几个问题：

怎样判断能否一直逃脱

根据题意，n个节点却只有n条边，所以图中只能且必须有一个环。
只有当b能到达环，且a不能在b到达环之前抓住b，b才能逃脱
这衍生出了几个问题：

  1. 如何找出环
  2. 如何判断b到环的距离
  3. 如何判断a能否在b到环上前抓住b

如何找出环

  • 利用并查集，在读取边的时候，如果发现两个顶点已经在同一个集合内了，则说明它们一定是环上的两个点
  • 从这环上的两个点其中之一出发，深度优先搜索dfs另一个点
  • 搜索到终点后，做一个全局标记，让函数返回上一层时记录下对应的点，得到所有环上的点
  • 最后将两个点连起来

如何判断b到环的距离

  • 利用两个数组，一个表示b到图中任意一点的距离，另一个表示节点是否访问过
  • 从b出发，广度优先搜索bfs，记录下沿途每一个点的距离，标记访问
  • 如果搜索到环上的点，记录距离和该点的编号（不妨设为c），停止；

如何判断a能在b到环上前抓住b

只要a到c的距离小于等于b，则a一定能抓住b，则同上：

  • 还是两个数组
  • 从a出发，bfs，记录距离和标记访问
  • 直到搜索到c
