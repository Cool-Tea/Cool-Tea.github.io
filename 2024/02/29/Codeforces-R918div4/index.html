C

Description

https://codeforces.com/contest/1915

Solution

如何快速检查平方数？二分在此发挥作用

每次二分检查mid是否为n的根，是就返回，否就根据mid的平方大小进行下一步二分

由此得到启发，所有需要试数/枚举的题目，不妨采用二分进行快速排查

E

Description

https://codeforces.com/contest/1915/problem/E

Solution

本题关键在于对题目公式进行变换

al+al+2+⋯+ar=al+1+al+3+⋯+ar−1a_l + a_{l+2} + \cdots + a_r = a_{l+1} + a_{l+3} + \cdots + a_{r-1} al​+al+2​+⋯+ar​=al+1​+al+3​+⋯+ar−1​

等价于

al−al+1+al+2−al+3+⋯−ar−1+ar=0a_l - a_{l+1} + a_{l+2} - a_{l+3} + \cdots - a_{r-1} + a_r = 0 al​−al+1​+al+2​−al+3​+⋯−ar−1​+ar​=0

于是我们只需要把偶数下标位置上变为相反数，这样这道题就等价于求是否存在子段和为0的子段

进一步转化，即前缀和中是否存在两个相等的元素，于是我们可以用一个set来维护前缀和，一旦发现元素重复，则说明存在

F

Description

https://codeforces.com/contest/1915/problem/F

Solution

本题思路比较简单，就是根据起点（或者终点）排序后对应每一个位置，看其后面（前面）的元素的终点（起点）比自己终点（起点）小（大）的个数

本质就是对一个区间 [l,r][l,r][l,r] 其包含的其他区间个数，最后求所有区间这个个数和

思路很简单，但是stl的set并不能做到在 O(log⁡N)O(\log N)O(logN) 的时间内返回比某个数大的元素个数，这会导致 O(N2)O(N^2)O(N2) 的复杂度

于是我们要么手搓一个红黑树，要么使用pbds下的tree

PBDS

Policy Based Data Structures

gnu下的内置数据结构，在<ext/pb_ds/>下，封装了hash, tree, trie, priority_queue

使用

#include <ext/pb_ds/assoc_container.hpp> //前置头文件
#include <ext/pb_ds/tree_policy.hpp> //用tree
#include <ext/pb_ds/hash_policy.hpp> //用hash
#include <ext/pb_ds/trie_policy.hpp> //用trie
#include <ext/pb_ds/priority_queue.hpp> //用priority_queue
using namespace __gnu_pbds;

tree中有个order_of_key函数，可以直接返回元素在集合中的位置（排序后），借助其可以实现快速计算比某个数大的元素个数

G

Description

https://codeforces.com/contest/1915/problem/G

Solution

乍一看挺像dijkstra算法，但是根据样例1，我们手算一下，发现我们可能需要花费额外时间去更远的城市换一个更快的单车，而这是dijkstra不允许的，因为dijkstra是贪心算法

我们把问题抽象出来，每一步状态可以用3个数描述， iii 表示现在所在的城市， sss 表示现在单车的slowness， ttt 表示距离出发花费的时间，故 (i,s,t)(i, s, t)(i,s,t) 描述了一个状态

状态间的转移如下

i→ji \rightarrow ji→j

s→min⁡(s,slowness[j])s \rightarrow \min (s, slowness[j])s→min(s,slowness[j])

t→t+w×st \rightarrow t + w \times st→t+w×s

其中，我们要令 ttt 最小，不妨如此定义：

t[i][s]t[i][s]t[i][s] 表示在 (i,s)(i, s)(i,s) 下的最小值

这样我们可以利用dijkstra算法快速得到最终答案

Details

  1. 最后dijkstra完了后要遍历s寻找t[n]中的最小值才是答案
  2. 注意数据大小，long long
