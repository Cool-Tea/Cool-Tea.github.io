1. Nanami and Toys Buying Problem

Description

给定 nnn 列，有无限行，第一行与第二行的物品价值用数组 a,ba, ba,b 给出，第二行以下的物品价值与第二行相同，给定 kkk 次机会取走，每次取必须取每列最上面的物品，问最大可取价值是多少

Solution

现在我们有几种策略，一是每次只拿表面价值最高的物品，二是找到第二行价值最大的物品，拿掉上面的，剩下全部拿该列的物品

但是我们可以通过几个简单的例子证明这些策略是错误的：

(1) 共拿k次
1 100 1000 10000
10000000000 100 1000 1000
(2) 共拿1次
3 4 5
10000 0 0

我们发现策略一没有考虑到第二行，导致不拿1下面的最大数；策略二没有考虑到第一行和k，如果次数有限则应该优先拿第一行最大的

所以我们要想到一个考虑第一行与第二行，同时兼顾k的策略

于是我们想，如果要获得最大的价值，我们应该考虑两个数组中的最大数，如果在第一行，则我们直接拿，如果在第二行，则我们要拿掉上面的才能拿到，此时还要考虑到k是否足够的问题，如果不够，则应该优先选第一行的次大数

但是我们看下面的例子：

共拿3次
0 1000 1001 1002
1003 0 0 0

这个例子中，如果按照我们的策略，则我们会选择 0 1003 1003，而实际上应该选择 1002 1001 1000，这是没有考虑到上面的数对整体的影响

那么我们换一种思路，我们能否通过暴力的方法解决呢

我们假设最终用于累加的列为第i列，则我们可以通过二分查找的方法找第一行中比第二行第i列数大的所有数，判断取完这些数后是否还有次数，如果有就看这些数里面有没有第i列的第一行数，有就直接取第二行，没有就取完第一行再取第二行，然后计算价值，最后取这些价值中的最大值即为解

而由于我们在查找过程中使用了二分，复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn) ，是在可接受的范围内

Details

  1. 本题需要特判最大值在第二行的情况，因为二分查找会查到第一行的最大值，但是最大值不取第一行最大值，而是直接取第二行的最大列

11. 安营扎寨

Description

求最大子矩阵和，但是要求子矩阵必须包含最中间的四个点

Solution

基于最大子段和，求最大子矩阵和升级为二维情况，这时需要考虑的因素变多

我们可能会想尝试二维情况下的动态规划，依照一维的情况，假设 dp[i][j]dp[i][j]dp[i][j] 是以 (i,j)(i,j)(i,j) 为右下端点的最大子矩阵和，但是在设计转移方程的时候发现需要左上角端点的信息，而且需要 (i−1,j),(i,j−1)(i-1,j),(i,j-1)(i−1,j),(i,j−1) 两个点的信息，这使得难以写出状态转移方程

其实更好的办法是将二维的问题转化为一维的问题，然后枚举求解

我们用 sum[i][j]sum[i][j]sum[i][j] 来表示第 jjj 列前 iii 行数的和，然后给定上下界 u,du,du,d ，转化为求 a[j]=sum[d][j]−sum[u−1][j]a[j] = sum[d][j] - sum[u-1][j]a[j]=sum[d][j]−sum[u−1][j] 这个数列的最大子段和（其实就是将 [(u,1),(d,m)][(u, 1), (d, m)][(u,1),(d,m)] 这个区域的列求和后作为数组求最大子段和）

我们只需要枚举 u,du,du,d 的组合就可以了

由于一维的最大子段和复杂度为 O(m)O(m)O(m) ，而枚举的复杂度为 O(n2)O(n^2)O(n2) ，所以本题做法的复杂度为 O(mn2)O(mn^2)O(mn2)

Details

  1. 本题会出现极端情况，即 m=1,n=105m = 1, n = 10^5m=1,n=105 此时需要掉换 m,nm,nm,n 位置，如此控制后，复杂度变为 O(mnmax⁡(m,n))O(mn \max (m, n))O(mnmax(m,n))

14. 小L的神奇修正带

Description

给定两个字符串 S,TS,TS,T 其中 ∣S∣≥∣T∣|S| \ge |T|∣S∣≥∣T∣ ，问 SSS 是否为 TTT 不断叠加覆盖形成的

Solution

首先我们要注意到一件事情：如果 SSS 是叠加而来，无论 TTT 怎么叠加，最后叠加的一定在最表面，即给定的 SSS 表面一定有至少一个 TTT

我们将其“撕掉”，但是剩下空位怎么处理呢？

我们并不知道底下的具体叠加情况，但是我们可以假设底下的叠加都是合法的，这些空位可以随便指定一个占位符，在这种情况下，我们继续查找“表面”的 TTT，只不过这一次，我们把占位符看成通配符，即这些字符匹配任何字符都是合法的

最后我们只需要检测 SSS 字符串是否全为占位符就可以判断答案

这样的复杂度为 O(mn)O(mn)O(mn) 大约为 5×1055 \times 10^55×105 的量级，所以不用担心会TLE

Details

  1. 由于我们是一层一层“撕”的，所以我们使用队列来完成这个任务
  2. 不要重复添加已经撕下的部分，即全是占位符是不允许的

17. 汽水装填

Description

给定一个数组的初始状态，再给定操作区间与操作数，把区间内小于操作数的数全部设置成为操作数，其余数不变，最后输出数组的最终状态

Solution

这种涉及到区间操作的一般有多种选择，比如线段树、树状数组等等，这里使用线段树

线段树使用二叉树结构，每个节点表示一段子区间，而把该节点的区间分成两半，左边区间赋给左子节点，右边的区间赋给右子节点，一直递归到区间中只有一个数

同时，每个节点含有一个标记，表示对整个区间的操作数，就本题而言，其表示区间中最大的操作数

以下是具体的线段树的操作过程：

给定一个区间 [l,r][l,r][l,r] 与操作数 bbb ，对于某一个节点来说有三种情况

  • 节点表示的区间与目标区间没有交集，这时直接返回
  • 节点表示的区间被包含在目标区间中，这时将其标记 mmm 设置为 max⁡(m,b)\max (m, b)max(m,b) ，然后返回
  • 节点表示的区间与目标区间相交，但是不是包含关系，这时需要递归到左右子节点进行操作，但是在调用左右节点前，需要把自己的标记 mmm “传”到左右子节点 mchild=max⁡(m,mchild)m_{child} = \max (m, m_{child})mchild​=max(m,mchild​) ，然后把自己的标记设0后再进行调用

之所以要“传”标记，是因为有可能左右节点区间的情况不一样，需要把标记传下去，让左右节点依据自己情况具体判断，同时父节点设0防止干扰

下面是查询线段树的操作过程：

给定查询区间 [l,r][l,r][l,r] （一个点也可以写成区间的形式），对于某一个节点依旧是上面的三种情况：

  • 没有交集，返回0
  • 包含关系，返回标记
  • 相交关系，“传标记”，查询左右节点，返回两者之间的最大值

最后输出的状态就是 max⁡(ai,mi)\max (a_i, m_i)max(ai​,mi​)

单次操作的时间复杂度为 O(log⁡n)O(\log n)O(logn) ，单次查询的复杂度为 O(log⁡n)O(\log n)O(logn)

Details

  1. 就本题而言，cin与cout需要关闭同步流，或者使用pringf与scanf，否则会TLE

6. Travel

Description

有向图中，定义 f(x,y,k)f(x,y,k)f(x,y,k) 为从 xxx 到 yyy 恰好经过 kkk 条边的走法个数，给定一个图，判断是否 ∃x,y\exist x, y∃x,y 使得 lim⁡k→+∞f(x,y,k)\lim\limits_{k\to+\infty}f(x,y,k)k→+∞lim​f(x,y,k) 发散

图中存在自环与多次自环

Solution

首先理解题意，什么叫 lim⁡k→+∞f(x,y,k)\lim\limits_{k\to+\infty}f(x,y,k)k→+∞lim​f(x,y,k) 发散？

很自然地，我们想到，如果 lim⁡k→+∞f(x,y,k)→∞\lim\limits_{k\to+\infty}f(x,y,k)\to\inftyk→+∞lim​f(x,y,k)→∞ ，则发散

但其实还有一种发散方式，当 k→+∞,f(x,y,k)k\to+\infty,f(x,y,k)k→+∞,f(x,y,k) 是周期性变化时，它也发散

说到周期性变化，如果有向图成环时，是不是 k→+∞,f(x,y,k)k\to+\infty,f(x,y,k)k→+∞,f(x,y,k) 就会周期变化呢？

我们考虑三种情况：

  • 多节点环：显然在环上的任意两点之间会发生周期性变化
  • 单节点多自环：直接取 x=yx = yx=y ，假设自环个数为 ccc ，则 f(x,x,k)=ckf(x,x,k) = c^kf(x,x,k)=ck ，则显然 k→+∞,f(x,y,k)→+∞k\to+\infty,f(x,y,k)\to+\inftyk→+∞,f(x,y,k)→+∞
  • 单节点单自环：此时情况比较复杂，我们下面讨论该情况

如果节点单自环，且它所能到达的路径中不存在另一个自环的节点，那么对于该节点而言，它与路径上任何的点之间的 fff 最终都会收敛到两者之间的通路数量

如果路径上存在另一个自环的节点，则对于这两个节点之间，由于可以自由选择在自身循环的次数，所以 k→+∞,f(x,y,k)→+∞k\to+\infty,f(x,y,k)\to+\inftyk→+∞,f(x,y,k)→+∞

所以我们得出结论：只有图中无多节点环，且自环的点之间不互达，这样的图才是收敛的，其余的都发散

16. 小L的方格问题

Description

给定一个 n×mn \times mn×m 的矩阵，每一行取出一个数求和，输出和中前k小的

Solution

首先意识到本题绝对不能暴力枚举，枚举量在 mnm^nmn 的量级，绝对TLE

由于是每一行取一个数，所以行内的顺序没有影响，不妨行内进行排序

如果 n=1n=1n=1 ，则直接输出前 kkk 个数

如果 n=2n=2n=2 ，此时 m2m^2m2 的量级可以枚举，也有线性的做法，下面讨论线性做法

对于行 a,ba, ba,b ，注意到

a[0]+b[0]<a[0]+b[1]<⋯<a[0]+b[i]<⋯<a[0]+b[m−1]a[0] + b[0] < a[0] + b[1] < \cdots <a[0] + b[i] < \cdots < a[0] + b[m-1] a[0]+b[0]<a[0]+b[1]<⋯<a[0]+b[i]<⋯<a[0]+b[m−1]

a[1]+b[0]<a[1]+b[1]<⋯<a[1]+b[i]<⋯<a[1]+b[m−1]a[1] + b[0] < a[1] + b[1] < \cdots <a[1] + b[i] < \cdots < a[1] + b[m-1] a[1]+b[0]<a[1]+b[1]<⋯<a[1]+b[i]<⋯<a[1]+b[m−1]

⋯\cdots ⋯

a[n−1]+b[0]<a[n−1]+b[1]<⋯<a[n−1]+b[i]<⋯<a[n−1]+b[m−1]a[n-1] + b[0] < a[n-1] + b[1] < \cdots <a[n-1] + b[i] < \cdots < a[n-1] + b[m-1] a[n−1]+b[0]<a[n−1]+b[1]<⋯<a[n−1]+b[i]<⋯<a[n−1]+b[m−1]

这么一个不等式矩阵，其枚举了所有的组合情况

于是我们就知道了这个矩阵每一行的最小值，但是我们并不知道这个最小值相对于其他行的值情况，这时我们就可以使用优先队列，把每一行最小值插入队列中，逐步弹出最小值，并且在弹出最小值的同时把该值在不等式矩阵中的后面一个数插入队列中，直到弹出 kkk 个数后停止。

如果 n>2n>2n>2 ，我们在线性做法上稍作改变

我们队列弹出 mmm 个数才停止，然后把这 mmm 个数作为新的一行代替原矩阵中的两行，也就是把两行压缩成一行，矩阵变为 (n−1)×m(n-1) \times m(n−1)×m ，重复操作直至矩阵降为一维，然后输出前 kkk 个数

Details

  1. 优先队列可以使用stl自带的容器，也可以使用算法库的make_heap
