Description

Solution

这道题最大的难点在于把lll与rrr数组的特殊性，以及aaa和bbb操作的特殊性弄清楚

lll & rrr

题目说给一个x夹在某个lll与rrr中间，意思就是lll与rrr是区间[l,r][l,r][l,r]的集合，而且这个集合拥有以下的特性：

  • l1l_1l1​对应字符串开头
  • rnr_nrn​对应字符串结尾
  • 两个区间是紧挨着的（lk+1=rk+1l_{k+1}=r_k+1lk+1​=rk​+1）

所以可见这个集合覆盖了整个字符串，而且两两之间不重合

aaa & bbb

题面给的aaa和bbb经过观察样例可以发现，aaa和bbb在某个区间内是左右对称的

思路

所以我们可以采取用数组记录操作的位置，然后在输出的时候根据记录来决定移动的方向和读取的顺序
数组如此记录arr[a]=b,arr[b]=aarr[a]=b, arr[b]=aarr[a]=b,arr[b]=a
读取如下：

  • 如果arr[i]=−1arr[i]=-1arr[i]=−1，则输出
  • 否则跳转至arr[i]arr[i]arr[i]的位置，按相反方向输出
  • 途中如果遇见arr[i]arr[i]arr[i]有值的情况，则跳转然后反向输出
  • 直到输出到跳转起始位置

由于&的性质，a≥a&ba \geq a \And ba≥a&b，所以可以认为在lll固定的情况下，f(l,r)f(l, r)f(l,r)关于r递减，可以看作是一个降序的数组
降序的数组怎么查最快？当然是二分查找！

思路

所以我们在读取数据后生成对应的arrarrarr，然后按照以上方法进行计算即可

Binary Search

Main Structure

二分查找一定有lll和rrr，然后根据条件不断二分

int l = begin, r = end;

    int mid = (l + r) / 2;

Details

根据情况不同，判断的条件也不同
这里假设arrarrarr是升序的 一共有两类情况：

  1. 查找大于（等于）kkk的最左的位置
  2. 查找小于（等于）kkk的最右的位置
Index Condition Statement1 Statement2
1 arr[mid] >(=) k r = mid l = mid + 1
2 arr[mid] <(=) k l = mid r = mid - 1

这里1、2的表述其实是一样的，因为查到了1，往左一位就是2的答案
而在实际中，应该尽量使用1而不是2，尽量避免l = mid, r = mid - 1这种情况出现，因为当r−l=1r-l=1r−l=1的时候，mid=lmid=lmid=l，这就会导致l=midl=midl=mid不断执行而不会结束循环，而1的写法就能避免这一点

还需要注意的一点，当将2的情况变为1的写法时，一定要注意边界结果，因为我们返回答案时会返回l−1l-1l−1，所以当答案是size−1size-1size−1时，会返回size−2size-2size−2
