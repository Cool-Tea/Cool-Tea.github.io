Newcoder: 小白月赛78

2023/12/22 Newcoder

C

题面

https://ac.nowcoder.com/acm/contest/65157/C

解题思路

本题的目的是找到第k小的数的最小值，其实就是通过以上规则生成k大小的升序数组arr，且保证数组内的任意两个数的和要么比arr[k]大，要么就是arr中的数，不可能有新的数能够插入到arr中
那我们的思路就是从小往大生成数组

暴力生成

对于已有的数组arr，我们直接将所有可能的和放入堆中，最后输出最小的那个，放到arr最后
有几个细节：

  • 在加入heap之前一定要检查是否重复
  • arr加入heap的堆顶元素也要检查是否重复

动态规划

前面暴力生成的时间复杂度为O(k!)，显然会超时
考虑数组中的每一个元素，其都是由给定的a和b生成而来，即

ai=ma+nba_i=ma+nb ai​=ma+nb

所以想要找到数组的下一个数，即找到

ak+1=min⁡(ai+aj)=min⁡((mi+mj)a+(ni+nj)b)>aka_{k+1}=\min(a_i+a_j)=\min((m_i+m_j)a+(n_i+n_j)b)>a_k ak+1​=min(ai​+aj​)=min((mi​+mj​)a+(ni​+nj​)b)>ak​

又有

min⁡(ai+aj)=min⁡(aj+a,aj+b)\min(a_i+a_j)=\min(a_j+a, a_j+b) min(ai​+aj​)=min(aj​+a,aj​+b)

故对于a和b，我们分别用两个指针去寻找它们最小临界的位置，即

ap1=min⁡(aj+a)>aka_{p_1} = \min(a_j+a) > a_k ap1​​=min(aj​+a)>ak​

ap2=min⁡(aj+b)>aka_{p_2} = \min(a_j+b) > a_k ap2​​=min(aj​+b)>ak​

然后再取两者的最小值作为数组的下一个数

有几个细节：

  • 一定是两个指针，不能只取最小值的指针，如a = 3，b = 10，在生成到{3, 6, 9, 10, 12, 13, 15, 16, 18, 19}时，如果按最小值指针，则会生成21，但是取10+10生成20，显然不对

题面

https://ac.nowcoder.com/acm/contest/65157/D

解题思路

法一

从简单入手，如果是满二叉树，则按树的高度H依次计数，假设有n个节点

n=2H−1n=2^H-1 n=2H−1

树高为i时有满二叉树数量为

ai=2H−i+1−1a_i=2^{H-i+1}-1 ai​=2H−i+1−1

求和有

sum=∑i=1Hai=2H+1−H−2sum = \sum_{i=1}^H a_i = 2^{H+1}-H-2 sum=i=1∑H​ai​=2H+1−H−2

如果增加k个节点(k < 2^H)，则对应的表为

k 1 2 3 ... 2H−12^{H}-12H−1
增加的值 1 3 4 ... 2H+1−H−22^{H+1}-H-22H+1−H−2
k的二进制 1 10 11 ... 111...111

观察发现增加的值可以这样计算

  • k的二进制中为1的位置pos
  • 计算2pos−12^{pos}-12pos−1
  • 所有的这些数加起来即为增加的值

法二

对于每一层而言，有aia_iai​个数
当只计算树高为1时，它们的贡献为aia_iai​
当只计算树高为2时，它们的贡献为ai/2a_i/2ai​/2
当只计算树高为3时，它们的贡献为ai/4a_i/4ai​/4
...
当只计算树高为k时，它们的贡献为ai/2k−1a_i/2^{k-1}ai​/2k−1
最后所有贡献加和即为该层的总贡献

细节处理

因为最后的答案可能很大所以要在每一步都要模109+110^9+1109+1

long long solve2(long long n) {
    int H = 1;
    long long sum = 0;
    while (n > 0) {
        long long t;
        if (n >= ((long long)1 << (H - 1))) {
            t = ((long long)1 << (H - 1));
        }
        else {
            t = n;
        }
        n -= t;
        while (t > 0) {
            sum += t % MOD; //not enough
            sum %= MOD; //the essence!!!
            t >>= 1;
        }
        H++;
    }
    return sum;
}

解题思路

法一

题意非常明确，思路按照题意来，唯一的难点是如何判断是否死循环
这里采用set，在每次操作完一轮后将x插入set中，如果重复则说明死循环

法二

按照法一来，会遇见时间复杂度过大的问题，从而导致RE
这里将次数分为两个部分，a轮+b步，先讨论一轮之后的情况，假设xkx_kxk​表示k轮后的x，由于

x=(x+a(i−1)mod(m))mod(n)x=(x+a_{(i-1)mod(m)})mod(n) x=(x+a(i−1)mod(m)​)mod(n)

可以认为

xk+1=(xk+∑i=1mai)mod(n)x_{k+1}=(x_k+\sum_{i=1}^ma_i)mod(n) xk+1​=(xk​+i=1∑m​ai​)mod(n)

故xkx_kxk​是模n的一个剩余类
可知，由抽屉原理，在n+1n+1n+1轮中，一定有两个x相等，故循环最多进行n轮，n轮中没有0则说明死循环
再讨论k步后的情况，假设k步后恰好x为0，则说明

x+(∑i=1kai)mod(n)=nx+(\sum_{i=1}^ka_i)mod(n)=n x+(i=1∑k​ai​)mod(n)=n

所以我们可以用一个map记录前缀和对n取模的值（如果有两个值相等，保留最小的下标），每轮只需要查询n−xn-xn−x在map中是否存在就行了

这道题会爆int，必须用long long
