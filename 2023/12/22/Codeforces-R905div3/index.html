2023/12/22 Codeforces

D

Description

https://codeforces.com/contest/1883/problem/D

Solution

题意让我们找到两个不重叠的子区间，所以本题的关键就在于如何快速判断有没有重叠的子区间
现在我们思考，两个区间不重叠的条件是什么？
即区间边界是分离的，左区间的右边界小于等于右区间的左边界
而题目要求只要找到一对，那我们可以取极端情况：
右边界最靠左的区间和左边界最靠右的区间，如果这两个区间不重叠，则满足题意，否则，没有其他区间不重叠（可以反证证明）

F

Description

https://codeforces.com/contest/1883/problem/F

Solution

题意要求找到不重复的子列的数量，但是不要求子列是连续的
本题关键在于如何找到不重复的子列，同D，两个子列不重复的条件是什么？不好判断，所以我们想两个子列重复的条件是什么？
两个子列的左右边界上的数相等
但是这是一个必要条件，我们如何补充条件使其充分？
如果子列的左边界左边没有左边界上的数，右边界右边没有右边界上的数，则该子列一定是不重复的
可以证明，这样取子列可以取完所有符合条件的子列
假设对于子列[l,r],∃α∈[0,l],β∈[r,n−1],a[α]=a[l],a[β]=a[r][l,r], \exist \alpha \in [0, l],\beta \in [r,n-1],a[\alpha]=a[l],a[\beta]=a[r][l,r],∃α∈[0,l],β∈[r,n−1],a[α]=a[l],a[β]=a[r]，则我们可以取出[l+1,r][l+1,r][l+1,r]和[l,r−1][l,r-1][l,r−1]的子列组成两个同样子列，矛盾！
所以我们有了一个贪心策略

  • 首先生成两个数列，分别记录该位置上的数是否是第一次出现和最后一次出现
  • 然后对第一次出现的数列求前缀和
  • 遍历每一个最后一次出现的数，将其对应的前缀和相加

本题分为两个难度，后一个是前一个的深化

G1

解题的关键就在于数列排序的任意性，既然排序是任意的，不妨将两个数列排序后再进行操作
由于我们想尽可能减少移除的元素个数，所以我们应该建立尽可能多的映射a[i]⇔b[j],a[i]<b[j]a[i] \lrArr b[j], a[i] < b[j]a[i]⇔b[j],a[i]<b[j]，由此我们有了一个贪心策略：

  • 找到数组结尾就结束

直观上看，因为b[j]b[j]b[j]越往后越大，跟田忌赛马一样，应该能对应更多的a[i]a[i]a[i]，于是我们有了G1的双指针+贪心的做法：

  • 两个指针分别指向a[0]a[0]a[0]和b[0]b[0]b[0]
  • 如果b[j]b[j]b[j]不是严格大于a[i]a[i]a[i]，就让bbb指针向后移动直到找到严格大于的数
  • 如果满足映射关系，则两个指针同时自增
